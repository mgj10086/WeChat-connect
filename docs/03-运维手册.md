# 运维手册

## 服务管理

| 操作 | 命令 |
|------|------|
| 启动 | `.\scripts\start-daemon.ps1` |
| 停止 | `.\scripts\stop-daemon.ps1` |
| 查看状态 | `.\scripts\status.ps1` |
| 查看日志 | 查看 daemon 窗口输出 |
| 重新登录 | `node src/dist/main.js setup` |

## 数据管理

数据存储在 `%USERPROFILE%\.wechat-claude-code\`：

| 路径 | 说明 | 备份频率 |
|------|------|---------|
| `accounts/` | 微信登录凭证 | 每次扫码后 |
| `sessions/` | 会话历史 | 按需 |
| `config.json` | 运行配置 | 修改后 |

备份：
```powershell
Copy-Item "$env:USERPROFILE\.wechat-claude-code" "D:\backups\wechat-claude-backup-$(Get-Date -Format 'yyyy-MM-dd')" -Recurse
```

## 故障排查

### 微信收不到回复

1. daemon 窗口是否还在运行？
2. 检查是否有错误输出
3. 运行 `.\scripts\status.ps1` 确认进程存在
4. 重新扫码：`node src/dist/main.js setup`
5. 重启 daemon

### daemon 启动报错

检查环境：
```powershell
node --version
claude --version
echo $env:ANTHROPIC_BASE_URL
echo $env:ANTHROPIC_AUTH_TOKEN
```

## 长期运行

要让 daemon 在后台持续运行（不依赖终端窗口）：

### 方案 A：Windows Task Scheduler（推荐）

1. 打开 Task Scheduler
2. 创建任务 → "用户登录时触发"
3. 操作：启动 `powershell.exe`
4. 参数：`-WindowStyle Hidden -File "D:\works\AIclass\agent\WeChat_claude\scripts\start-daemon.ps1"`
5. 勾选"不管用户是否登录都要运行"

### 方案 B：nssm（Windows Service Wrapper）

```powershell
# 安装 nssm
choco install nssm

# 注册服务
nssm install WeChatClaudeCode "C:\Program Files\nodejs\node.exe" "src/dist/main.js"

# 启动服务
nssm start WeChatClaudeCode
```
